## Why Designer Mode?

<CardGrid>
  <Card title="Instant feedback loop" icon="rocket">
    See changes in seconds, not days. No more waiting for a developer to interpret a Figma mockup and ship it.
  </Card>
  <Card title="Zero translation loss" icon="approve-check-circle">
    Point at the exact element and say what you want. No ambiguity, no misinterpretation, no back-and-forth.
  </Card>
  <Card title="No code skills needed" icon="pencil">
    Work visually in the browser. The AI agent handles finding the right file, component, and prop to change.
  </Card>
  <Card title="Works with the real product" icon="laptop">
    Not a mockup or prototype. Tweak the actual running app with real data and real components.
  </Card>
  <Card title="AI does the heavy lifting" icon="star">
    The agent figures out which file, which component, and which prop to change. Just describe what you want in plain language.
  </Card>
  <Card title="Designer opens the PR, dev reviews" icon="github">
    The designer drives the changes and opens the pull request. Developers review the code — not interpret mockups.
  </Card>
</CardGrid>
